### Command

The definition of a command. A command is a user-defined operation which can take arguments and returns an output. The semantics of a command are opaque to the Open DD specification.

**Properties**

| Name | Type | Required | Description |
| ----- | ----- | ----- | ----- |
| `name` | `string` | Yes | The name of the command. |
| `outputType` | `string` | Yes | The return type of the command. |
| `arguments` | [`array`](#argumentdefinition) | No | The list of arguments accepted by this command. Defaults to no arguments. |
| `source` | [`object`](#commandsource) | No | The source configuration for this command. |
| `graphql` | [`object`](#commandgraphqldefinition) | No | Configuration for how this command should appear in the GraphQL schema. |
| `description` | `string` | No | The description of the command. Gets added to the description of the command's root field in the graphql schema. |


**Example:**

```yaml
name: get_latest_article
outputType: commandArticle
arguments: []
source:
  dataConnectorName: data_connector
  dataConnectorCommand:
    function: latest_article
  typeMapping:
    commandArticle:
      fieldMapping:
        article_id:
          column: id
  argumentMapping: {}
graphql:
  rootFieldName: getLatestArticle
  rootFieldKind: Query
description: Get the latest article

```


#### ArgumentDefinition

The definition of an argument for a field, command, or model.



**Properties**

| Name | Type | Required | Description |
| ----- | ----- | ----- | ----- |
| `name` | `string` | Yes | N/A |
| `type` | `string` | Yes | A reference to an Open DD type including nullable values and arrays. Suffix '!' to indicate a non-nullable reference, and wrap in '[]' to indicate an array. Eg: '[String!]!' is a non-nullable array of non-nullable strings. |
| `description` | `string` | No | N/A |


#### CommandSource

Description of how a command maps to a particular data connector



**Properties**

| Name | Type | Required | Description |
| ----- | ----- | ----- | ----- |
| `dataConnectorName` | `string` | Yes | The name of the data connector backing this command. |
| `dataConnectorCommand` | [`object`](#dataconnectorcommand) | Yes | The function/procedure in the data connector that backs this command. |
| `typeMapping` | `object` | No | How the various types used in this command correspond to entities in the data connector. |
| `argumentMapping` | `object` | No | Mapping from command argument names to data connector table argument names. |


**Example:**

```yaml
dataConnectorName: data_connector
dataConnectorCommand:
  function: latest_article
typeMapping: {}
argumentMapping: {}

```


#### CommandGraphQlDefinition

The definition of how a command should appear in the GraphQL API.



**Properties**

| Name | Type | Required | Description |
| ----- | ----- | ----- | ----- |
| `rootFieldName` | `string` | Yes | The name of the graphql root field to use for this command. |
| `rootFieldKind` | `string` | Yes | Whether to put this command in the Query or Mutation root of the GraphQL API. |


**Example:**

```yaml
rootFieldName: getLatestArticle
rootFieldKind: Query

```


#### TypeReference

A reference to an Open DD type including nullable values and arrays.
Suffix '!' to indicate a non-nullable reference, and wrap in '[]' to indicate an array.
Eg: '[String!]!' is a non-nullable array of non-nullable strings.
