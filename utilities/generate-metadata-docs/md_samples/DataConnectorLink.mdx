### DataConnectorLink

Definition of a data connector, used to bring in sources of data and connect them to OpenDD models and commands.

**Properties**

| Name | Type | Required | Description |
| ----- | ----- | ----- | ----- |
| `name` | `string` | Yes | The name of the data connector. |
| `url` | [`object`](#dataconnectorurlv1) | Yes | The url(s) to access the data connector. |
| `headers` | `object` | No | Key value map of HTTP headers to be sent with each request to the data connector. |
| `schema` | [`object`](#versionedschemaandcapabilities) | Yes | The schema of the data connector. This schema is used as the source of truth when serving requests and the live schema of the data connector is not looked up. |


**Example:**

```yaml
name: data_connector
url:
  singleUrl:
    value: http://data_connector:8100
headers: {}
schema:
  version: v0.1
  schema:
    scalar_types: {}
    object_types: {}
    collections: []
    functions: []
    procedures: []
  capabilities:
    version: 0.1.0
    capabilities:
      query:
        variables: {}
      mutation: {}

```


**Properties**

| Name | Type | Required | Description |
| ----- | ----- | ----- | ----- |
| `singleUrl` | object | Yes | Either a literal string or a reference to a Hasura secret |
| `readWriteUrls` | object | Yes | N/A |
