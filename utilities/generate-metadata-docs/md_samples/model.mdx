### Model

The definition of a data model. A data model is a collection of objects of a particular type. Models can support one or
more CRUD operations.

**Properties**

| Name               | Type                                | Required | Description                                                                                     |
| ------------------ | ----------------------------------- | -------- | ----------------------------------------------------------------------------------------------- |
| `name`             | `string`                            | Yes      | The name of the data model.                                                                     |
| `objectType`       | `string`                            | Yes      | The type of the objects of which this model is a collection.                                    |
| `globalIdSource`   | `boolean`                           | No       | Whether this model should be used as the global ID source for all objects of its type.          |
| `arguments`        | [`array`](#argumentdefinition)      | No       | A list of arguments accepted by this model. Defaults to no arguments.                           |
| `source`           | [`object`](#modelsource)            | No       | The source configuration for this model.                                                        |
| `filterableFields` | [`array`](#filterablefield)         | Yes      | A list of fields that can be used to filter the objects in this model.                          |
| `orderableFields`  | [`array`](#orderablefield)          | Yes      | A list of fields that can be used to order the objects in this model.                           |
| `graphql`          | [`object`](#modelgraphqldefinition) | No       | Configuration for how this model should appear in the GraphQL schema.                           |
| `description`      | `string`                            | No       | The description of the model. Gets added to the description of the model in the graphql schema. |

**Example:**

```yaml
name: Articles
objectType: article
globalIdSource: true
arguments: []
source:
  dataConnectorName: data_connector
  collection: articles
  typeMapping:
    article:
      fieldMapping:
        article_id:
          column: id
        title:
          column: title
        author_id:
          column: author_id
  argumentMapping: {}
filterableFields:
  - fieldName: article_id
    operators:
      enableAll: true
  - fieldName: title
    operators:
      enableAll: true
  - fieldName: author_id
    operators:
      enableAll: true
orderableFields:
  - fieldName: article_id
    orderByDirections:
      enableAll: true
  - fieldName: title
    orderByDirections:
      enableAll: true
  - fieldName: author_id
    orderByDirections:
      enableAll: true
graphql:
  selectUniques:
    - queryRootField: ArticleByID
      uniqueIdentifier:
        - article_id
      description: Description for the select unique ArticleByID
  selectMany:
    queryRootField: ArticleMany
    description: Description for the select many ArticleMany
  argumentsInputType: null
  filterExpressionType: Article_Where_Exp
  orderByExpressionType: Article_Order_By
description: Description for the model Articles
```

#### ArgumentDefinition

The definition of an argument for a field, command, or model.

**Properties**

| Name          | Type     | Required | Description                                                                                                                                                                                                                    |
| ------------- | -------- | -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `name`        | `string` | Yes      | N/A                                                                                                                                                                                                                            |
| `type`        | `string` | Yes      | A reference to an Open DD type including nullable values and arrays. Suffix '!' to indicate a non-nullable reference, and wrap in '[]' to indicate an array. Eg: '[String!]!' is a non-nullable array of non-nullable strings. |
| `description` | `string` | No       | N/A                                                                                                                                                                                                                            |

#### ModelSource

Description of how a model maps to a particular data connector

**Properties**

| Name                | Type     | Required | Description                                                                            |
| ------------------- | -------- | -------- | -------------------------------------------------------------------------------------- |
| `dataConnectorName` | `string` | Yes      | The name of the data connector backing this model.                                     |
| `collection`        | `string` | Yes      | The collection in the data connector that backs this model.                            |
| `typeMapping`       | `object` | No       | How the various types used in this model correspond to entities in the data connector. |
| `argumentMapping`   | `object` | No       | N/A                                                                                    |

**Example:**

```yaml
dataConnectorName: data_connector
collection: articles
typeMapping:
  article:
    fieldMapping:
      article_id:
        column: id
      title:
        column: title
      author_id:
        column: author_id
argumentMapping: {}
```

#### ModelGraphQlDefinition

The definition of how a model appears in the GraphQL API.

**Properties**

| Name                    | Type                                      | Required | Description                                                                                                                                    |
| ----------------------- | ----------------------------------------- | -------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| `selectUniques`         | [`array`](#selectuniquegraphqldefinition) | Yes      | For each select unique defined here, a query root field is added to the GraphQL API that can be used to select a unique object from the model. |
| `selectMany`            | [`object`](#selectmanygraphqldefinition)  | No       | Select many configuration for a model adds a query root field to the GraphQl API that can be used to retrieve multiple objects from the model. |
| `argumentsInputType`    | `string`                                  | No       | The type name of the input type used to hold the arguments of the model.                                                                       |
| `filterExpressionType`  | `string`                                  | No       | The type name of the filter boolean expression input type.                                                                                     |
| `orderByExpressionType` | `string`                                  | No       | The type name of the order by expression input type.                                                                                           |

**Example:**

```yaml
selectUniques:
  - queryRootField: ArticleByID
    uniqueIdentifier:
      - article_id
    description: Description for the select unique ArticleByID
selectMany:
  queryRootField: ArticleMany
  description: Description for the select many ArticleMany
argumentsInputType: null
filterExpressionType: Article_Where_Exp
orderByExpressionType: Article_Order_By
```

#### TypeReference

A reference to an Open DD type including nullable values and arrays. Suffix '!' to indicate a non-nullable reference,
and wrap in '[]' to indicate an array. Eg: '[String!]!' is a non-nullable array of non-nullable strings.

#### SelectUniqueGraphQlDefinition

The definition of the GraphQL API for selecting a unique row/object from a model.

**Properties**

| Name               | Type                  | Required | Description                                                                                                                                                             |
| ------------------ | --------------------- | -------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `queryRootField`   | `string`              | Yes      | The name of the query root field for this API.                                                                                                                          |
| `uniqueIdentifier` | [`array`](#fieldname) | Yes      | A set of fields which can uniquely identify a row/object in the model.                                                                                                  |
| `description`      | `string`              | No       | The description of the select unique graphql definition of the model. Gets added to the description of the select unique root field of the model in the graphql schema. |

#### SelectManyGraphQlDefinition

**Properties**

| Name             | Type     | Required | Description                                                                                                                                                         |

| `queryRootField` | `string` | Yes      | The name of the query root field for this API.                                                                                                                      |
| `description`    | `string` | No       | The description of the select many graphql definition of the model. Gets added to the description of the select many root field of the model in the graphql schema. |
