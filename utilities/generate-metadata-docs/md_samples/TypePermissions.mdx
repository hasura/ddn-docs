### TypePermissions

Definition of permissions for an OpenDD type.

**Properties**

| Name | Type | Required | Description |
| ----- | ----- | ----- | ----- |
| `typeName` | `string` | Yes | The name of the type for which permissions are being defined. Must be an object type. |
| `permissions` | [`array`](#typepermission) | Yes | A list of type permissions, one for each role. |


**Example:**

```yaml
typeName: article
permissions:
  - role: admin
    output:
      allowedFields:
        - article_id
        - author_id
        - title
  - role: user
    output:
      allowedFields:
        - article_id
        - author_id

```


#### TypePermission

Defines permissions for a particular role for a type.



**Properties**

| Name | Type | Required | Description |
| ----- | ----- | ----- | ----- |
| `role` | `string` | Yes | The role for which permissions are being defined. |
| `output` | [`object`](#typeoutputpermission) | No | Permissions for this role when this type is used in an output context. If null, this type is inaccessible for this role in an output context. |


**Example:**

```yaml
role: user
output:
  allowedFields:
    - article_id
    - author_id

```


#### TypeOutputPermission

Permissions for a type for a particular role when used in an output context.



**Properties**

| Name | Type | Required | Description |
| ----- | ----- | ----- | ----- |
| `allowedFields` | [`array`](#fieldname) | Yes | Fields of the type that are accessible for a role |
