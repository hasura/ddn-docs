### Relationship

Definition of a relationship on an OpenDD type which allows it to be extended with related models or commands.

**Properties**

| Name | Type | Required | Description |
| ----- | ----- | ----- | ----- |
| `name` | `string` | Yes | The name of the relationship. |
| `source` | `string` | Yes | The source type of the relationship. |
| `target` | [`object`](#relationshiptarget) | Yes | The target of the relationship. |
| `mapping` | [`array`](#relationshipmapping) | Yes | The mapping configuration of source to target for the relationship. |
| `description` | `string` | No | The description of the relationship. Gets added to the description of the relationship in the graphql schema. |


**Example:**

```yaml
name: Articles
source: author
target:
  model:
    name: Articles
    namespace: null
    subgraph: null
    relationshipType: Array
mapping:
  - source:
      fieldPath:
        - fieldName: author_id
    target:
      modelField:
        - fieldName: author_id
description: Articles written by an author

```


#### RelationshipTarget

The target for a relationship.



**This object must have exactly one of the following:**

| Name | Type | Required | Description |
| ----- | ----- | ----- | ----- |
| `model` | `object` | No | The target model for a relationship. |
| `command` | `object` | No | The target command for a relationship. |


**Example:**

```yaml
model:
  name: Articles
  namespace: null
  subgraph: null
  relationshipType: Array

```


#### RelationshipMapping

Definition of a how a particular field in the source maps to a target field or argument.



**Properties**

| Name | Type | Required | Description |
| ----- | ----- | ----- | ----- |
| `source` | [`object`](#relationshipmappingsource) | Yes | The source configuration for this relationship mapping. |
| `target` | [`object`](#relationshipmappingtarget) | Yes | The target configuration for this relationship mapping. |


**Example:**

```yaml
source:
  fieldPath:
    - fieldName: author_id
target:
  modelField:
    - fieldName: author_id

```


#### ModelRelationshipTarget

The target model for a relationship.



**Properties**

| Name | Type | Required | Description |
| ----- | ----- | ----- | ----- |
| `name` | `string` | Yes | The name of the data model. |
| `namespace` | `string` | No |  |
| `subgraph` | `string` | No | The subgraph of the target model. Defaults to the current subgraph. |
| `relationshipType` | [`object`](#relationshiptype) | Yes | Type of the relationship - object or array. |


#### CommandRelationshipTarget

The target command for a relationship.



**Properties**

| Name | Type | Required | Description |
| ----- | ----- | ----- | ----- |
| `name` | `string` | Yes | The name of the command. |
| `subgraph` | `string` | No | The subgraph of the target command. Defaults to the current subgraph. |


#### RelationshipMappingSource

The source configuration for a relationship mapping.



**This object must have exactly one of the following:**

| Name | Type | Required | Description |
| ----- | ----- | ----- | ----- |
| `SourceValue` | `object` | No | An expression which evaluates to a value that can be used in permissions. |
| `SourceField` | `array` | No |  |


#### RelationshipMappingTarget

The target configuration for a relationship mapping.



**This object must have exactly one of the following:**

| Name | Type | Required | Description |
| ----- | ----- | ----- | ----- |
| `TargetArgument` | `object` | No |  |
| `TargetModelField` | `array` | No |  |


#### RelationshipType

Type of the relationship.



**This object must have exactly one of the following:**

| Name | Type | Required | Description |
| ----- | ----- | ----- | ----- |
| `Object` | `string` | No | Select one related object from the target. |
| `Array` | `string` | No | Select multiple related objects from the target. |


#### ValueExpression

An expression which evaluates to a value that can be used in permissions.



**This object must have exactly one of the following:**

| Name | Type | Required | Description |
| ----- | ----- | ----- | ----- |
| `Literal` | `object` | No |  |
| `SessionVariable` | `string` | No | Used to represent the name of a session variable, like "x-hasura-role". |
