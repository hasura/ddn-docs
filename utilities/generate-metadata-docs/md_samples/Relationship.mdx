### Relationship

Definition of a relationship on an OpenDD type which allows it to be extended with related models or commands.

**Properties**

| Name | Type | Required | Description |
| ----- | ----- | ----- | ----- |
| `name` | `string` | Yes | The name of the relationship. |
| `source` | `string` | Yes | The source type of the relationship. |
| `target` | [`object`](#relationshiptarget) | Yes | The target of the relationship. |
| `mapping` | [`array`](#relationshipmapping) | Yes | The mapping configuration of source to target for the relationship. |
| `description` | `string` | No | The description of the relationship. Gets added to the description of the relationship in the graphql schema. |


**Example:**

```yaml
name: Articles
source: author
target:
  model:
    name: Articles
    namespace: null
    subgraph: null
    relationshipType: Array
mapping:
  - source:
      fieldPath:
        - fieldName: author_id
    target:
      modelField:
        - fieldName: author_id
description: Articles written by an author

```


#### RelationshipMapping

Definition of a how a particular field in the source maps to a target field or argument.



**Properties**

| Name | Type | Required | Description |
| ----- | ----- | ----- | ----- |
| `source` | [`object`](#relationshipmappingsource) | Yes | The source configuration for this relationship mapping. |
| `target` | [`object`](#relationshipmappingtarget) | Yes | The target configuration for this relationship mapping. |


**Example:**

```yaml
source:
  fieldPath:
    - fieldName: author_id
target:
  modelField:
    - fieldName: author_id

```
