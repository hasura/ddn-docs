# Compatibility Config

## Introduction

The `CompatibilityConfig` object is a metadata object that defines the compatibility configuration of the Hasura
metadata. The `date` field in the `CompatibilityConfig` object specifies the date after which any backwards incompatible
changes made to Hasura DDN won't impact the metadata.

**The CompatibilityConfig object belongs to the supergraph** and can be defined once across your supergraph in any
subgraph of your choice.

By default, all projects are created with a default `CompatibilityConfig` object in the **globals** subgraph.

## Metadata structure

### v2_CompatibilityConfig {#v2_compatibilityconfig-v2_compatibilityconfig}

The compatibility configuration of the Hasura metadata.

| Key    | Value                 | Required | Description                                                                                      |
| ------ | --------------------- | -------- | ------------------------------------------------------------------------------------------------ |
| `kind` | `CompatibilityConfig` | true     |                                                                                                  |
| `date` | string                | true     | Any backwards incompatible changes made to Hasura DDN after this date won't impact the metadata. |
