# Metadata Operations

Codemods to help refactor your metadata.

## Rename GraphQL root fields and type name prefixes

Newly generated metadata uses prefixes set in the [subgraph.yaml generator configuration.](/supergraph-modeling/build-configs#subgraph-subgraphgeneratorconfig)

This codemod will rename prefixes in already generated metadata. Consequently, it can also be used to add or remove prefixes altogether.

The from prefix will be stripped if provided, and the new prefix will be added. If the new prefix is already present, it will not be reapplied.

Examples

```bash
# Add root field and type name prefixes to the subgraph set in the context
ddn codemod rename-graphql-prefixes --graphqlRootField 'app_' --graphqlTypeName 'App_'

# Change the root field prefix for the specified subgraph
ddn codemod rename-graphql-prefixes --subgraph app/subgraph.yaml --graphqlRootField 'foo_' --fromGraphqlRootField 'app_'
```
