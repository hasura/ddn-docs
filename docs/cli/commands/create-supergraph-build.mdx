# Hasura DDN CLI: ddn create supergraph-build 

## Synopsis
See docs here: https://hasura.io/docs/3.0/ci-cd/builds/.

```bash
$ ddn create supergraph-build --help
See docs here: https://hasura.io/docs/3.0/ci-cd/builds/

Usage:
  ddn create supergraph-build [flags]

Aliases:
  supergraph-build, SupergraphBuild, spgb, sb

Examples:
 # Create a Supergraph Build with the default supergraphManifest (hasura.yaml present in the current working directory)
   ddn create supergraph-build

 # Create a Supergraph Build with the default supergraphManifest and add additional ENV vars (ENV var name should be prefixed with 'supergraph' or with the name of the subgraph followed by a dot(.))
   ddn create supergraph-build -e supergraph.JWT_TOKEN=myToken -e app.CONNECTOR_URL=https://host:port

 # Create a Supergraph Build with an alternate supergraphManifest (hasura.yaml is not present in the current working directory)
   ddn create supergraph-build --supergraph-manifest prod.supergraph.hml --dir <path-to-hasura.yaml-file> --description "overridden supergraphManifest"

 # Create a Supergraph Build with metadata.json file (Mode: Replace)
   ddn create supergraph-build --mode replace --project <project-name> --metadata-from-file <path-to-metadata.json-file> --environment dev --description "no supergraph manifest build"

Flags:
  -d, --description string          (Optional) description about the build
      --dry-run                     Prints the metadata used to create the build. No build artifact is generated
  -e, --env strings                 env vars. ENV var name should be prefixed with 'supergraph' or with the name of the subgraph followed by a dot(.)
      --environment string          Environment with which to create the build. To be used when building with metadata JSON only. When provided, --mode, --project and --metadata-from-file must also be provided
  -h, --help                        help for supergraph-build
      --metadata-from-file string   Path to the metadata JSON file. To be used when building with metadata JSON only. When provided, --mode, --project and --environment must also be provided
      --mode string                 Can be patch or replace. To be used when building with metadata JSON only. When provided, --metadata-from-file, --project and --environment must also be provided
      --patch-on-build string       Override the patch on build value provided in the supergraph manifest, only valid if the mode is patch. Takes the build version, 'applied' or 'latest' as accepted values

Global Flags:
      --dir string                   DDN project directory in which hasura.yaml is present (default ".")
      --log-level string             Log level. accepts DEBUG, WARN, INFO, ERROR, FATAL (default "INFO")
      --no-prompt                    Set this flag to no prompt for anything, the command fails if any input is missing
  -o, --out string                   Format in which the output should be printed, accepts table, json and yaml (default "table")
  -p, --project string               DDN Project Name
  -s, --supergraph-manifest string   Supergraph manifest to use. (Optional. DefaultSupergraphManifest used  if not provided)
```
