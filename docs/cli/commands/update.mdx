# Hasura DDN CLI: ddn update 

## Synopsis
Update objects in the local metadata.

```bash
$ ddn update --help
Update objects in the local metadata

Usage:
  ddn update [command]

Examples:
 # Update a connector's BuildContext using the ConnectorManifest defined in SupergraphManifest
   ddn update connector-manifest <connector-name> --subgraph <subgraph-name>

 # Update a data connector's DataConnectorLink using the running connector URL specified in the DataConnectorLink object
   ddn update connector-link <connector-name> --subgraph <subgraph-name>


Available Commands:
  command            Update a Command in the local metadata
  connector-link     Updates the schema of a DataConnectorLink
  connector-manifest Update a connector's BuildContext
  model              Update a Model in the local metadata

Flags:
  -h, --help   help for update

Global Flags:
      --dir string                   DDN Project directory in which hasura.yaml is present (default ".")
      --log-level string             Log level. Can be DEBUG, WARN, INFO, ERROR, or FATAL. (default "INFO")
      --no-prompt                    Do not prompt for required but missing flags
  -o, --out string                   Output format. Can be table, json or yaml. (default "table")
  -p, --project string               DDN Project name
  -s, --supergraph-manifest string   SupergraphManifest to use.

Use "ddn update [command] --help" for more information about a command.
```
