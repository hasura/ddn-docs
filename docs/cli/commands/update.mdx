# Hasura DDN CLI: ddn update 

## Synopsis
Update Objects in the local metadata.

```bash
$ ddn update --help
Update Objects in the local metadata

Usage:
  ddn update [command]

Examples:
 # Update connector build context using the connector-manifest specified in the current Supergraphmanifest
   ddn update connectormanifest <connector-name> --subgraph app

 # Update DataConnectorLink using the running connector URL specified in the ConnectorLink File
   hasura3 update connectorlink <connector-name> --subgraph app

 # Update DataConnectorLink using the Deployment ID returned by the 'ddn build connectormanifest' command
   hasura3 update connectorlink <connector-name> --subgraph app --deployment-id <deployment-id>


Available Commands:
  connector-link     Updates the NDC schema of the connector link
  connector-manifest update the connector build context using the manifest

Flags:
  -h, --help   help for update

Global Flags:
      --dir string                   DDN project directory in which hasura.yaml is present (default ".")
      --log-level string             Log level. accepts DEBUG, WARN, INFO, ERROR, FATAL (default "INFO")
      --no-prompt                    Set this flag to no prompt for anything, the command fails if any input is missing
  -o, --out string                   Format in which the output should be printed, accepts table, json and yaml (default "table")
  -p, --project string               DDN Project Name
  -s, --supergraph-manifest string   Supergraph manifest to use. (Optional. DefaultSupergraphManifest used  if not provided)

Use "ddn update [command] --help" for more information about a command.
```
