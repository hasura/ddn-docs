# Hasura DDN CLI: ddn update 

## Synopsis
Update objects in the local metadata.

```bash
$ ddn update --help
Update objects in the local metadata

Usage:
  ddn update [command]

Examples:
 # Update a connector's BuildContext using the ConnectorManifest defined in SupergraphManifest
   ddn update connector-manifest <connector-name> --subgraph <subgraph-name>

 # Update a data connector's ConnectorLink using the running connector URL specified in the ConnectorLink object
   ddn update connector-link <connector-name> --subgraph <subgraph-name>

 # Update a data connector's ConnectorLink using the deployment ID returned by the 'ddn build connector-manifest' command
   ddn update connector-link <connector-name> --subgraph <subgraph-name> --deployment-id <deployment-id>


Available Commands:
  connector-link     Updates the schema of a ConnectorLink
  connector-manifest Update a connector's BuildContext

Flags:
  -h, --help   help for update

Global Flags:
      --dir string                   DDN project directory in which hasura.yaml is present (default ".")
      --log-level string             Log level. accepts DEBUG, WARN, INFO, ERROR, FATAL (default "INFO")
      --no-prompt                    Set this flag to no prompt for anything, the command fails if any input is missing
  -o, --out string                   Format in which the output should be printed, accepts table, json and yaml (default "table")
  -p, --project string               DDN Project Name
  -s, --supergraph-manifest string   SupergraphManifest to use. (Optional. DefaultSupergraphManifest used  if not provided)

Use "ddn update [command] --help" for more information about a command.
```
