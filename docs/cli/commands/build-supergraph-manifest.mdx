# Hasura DDN CLI: ddn build supergraph-manifest 

## Synopsis
See docs here: https://hasura.io/docs/3.0/ci-cd/builds/.

```bash
$ ddn build supergraph-manifest --help
See docs here: https://hasura.io/docs/3.0/ci-cd/builds/

Usage:
  ddn build supergraph-manifest [flags]

Examples:
 # Create a SupergraphBuild using the default SupergraphManifest defined in hasura.yaml
   ddn build supergraph-manifest

 # Create a SupergraphBuild with an alternate SupergraphManifest
   ddn build supergraph-manifest --supergraph-manifest prod.supergraph.hml --description "build for prod env"

 # Create a SupergraphBuild with specified ENV vars. (ENV var names should be prefixed with 'supergraph' or with the name of the subgraph followed by a dot(.))
   ddn build supergraph-manifest -e supergraph.JWT_TOKEN=myToken -e app.CONNECTOR_URL=https://host:port

 # Create a SupergraphBuild if hasura.yaml is not present in the current working directory
   ddn build supergraph-manifest --dir <path-to-hasura.yaml-file>

 # Create a SupergraphBuild with metadata.json file (Mode: Replace)
   ddn build supergraph-manifest --mode replace --project <project-name> --metadata-from-file <path-to-metadata.json-file> --environment dev --description "no SupergraphManifest build"


Flags:
  -d, --description string          (Optional) description about the build
      --dry-run                     Prints the metadata used to create the build. No build artifact is generated
  -e, --env strings                 env vars. ENV var name should be prefixed with 'supergraph' or with the name of the Subgraph followed by a dot(.)
      --environment string          Environment with which to create the build
  -h, --help                        help for supergraph-manifest
      --metadata-from-file string   Path to the metadata JSON file. To be used when building with metadata JSON only. When provided, --mode, --project and --environment must also be provided
      --mode string                 Can be patch or replace. To be used when building with metadata JSON only. When provided, --metadata-from-file, --project and --environment must also be provided
      --patch-on-build string       Override the patch on build value provided in the SupergraphManifest, only valid if the mode is patch. Takes the build version, 'applied' or 'latest' as accepted values

Global Flags:
      --dir string                   DDN project directory in which hasura.yaml is present (default ".")
      --log-level string             Log level. accepts DEBUG, WARN, INFO, ERROR, FATAL (default "INFO")
      --no-prompt                    Set this flag to no prompt for anything, the command fails if any input is missing
  -o, --out string                   Format in which the output should be printed, accepts table, json and yaml (default "table")
  -p, --project string               DDN Project Name
  -s, --supergraph-manifest string   SupergraphManifest to use. (Optional. DefaultSupergraphManifest used  if not provided)
```
