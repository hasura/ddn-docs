# Hasura DDN CLI: ddn build supergraph-manifest

## Synopsis
See docs here: https://hasura.io/docs/3.0/ci-cd/builds/

```bash
$ ddn build supergraph-manifest --help
See docs here: https://hasura.io/docs/3.0/ci-cd/builds/

Usage:
  ddn build supergraph-manifest [flags]

Aliases:
  supergraph-manifest, sm, SupergraphManifest

Examples:
 # Create a SupergraphBuild using the default SupergraphManifest defined in hasura.yaml
   ddn build supergraph-manifest

 # Create a SupergraphBuild with an alternate SupergraphManifest
   ddn build supergraph-manifest --supergraph-manifest prod --description "build for prod"

 # Create a SupergraphBuild with specified ENV vars. (Environment variable names should be prefixed with 'supergraph' or with the name of the subgraph followed by a dot(.))
   ddn build supergraph-manifest -e supergraph.JWT_TOKEN=myToken -e app.CONNECTOR_URL=https://host:port

 # Create a SupergraphBuild if hasura.yaml is not present in the current working directory
   ddn build supergraph-manifest --dir <path-to-hasura.yaml-file>

 # Create a SupergraphBuild with metadata.json file
   ddn build supergraph-manifest --metadata-from-file <path-to-metadata.json-file> --description "build using metadata file"


Flags:
  -d, --description string   (Optional) description about the build
      --dry-run              Prints the metadata JSON used to create the SupergraphBuild. No SupergraphBuild artifact is generated
  -e, --env strings          Set environment variables. Environment variable name should be prefixed with 'supergraph' or with the name of the Subgraph followed by a dot(.)
  -h, --help                 help for supergraph-manifest

Global Flags:
      --dir string                   DDN Project directory in which hasura.yaml is present (default ".")
      --log-level string             Log level. Can be DEBUG, WARN, INFO, ERROR, or FATAL. (default "INFO")
      --no-prompt                    Do not prompt for required but missing flags
      --out string                   Output format. Can be table, json or yaml. (default "table")
  -p, --project string               DDN Project name
      --supergraph-manifest string   SupergraphManifest to use
```
