# Hasura DDN CLI: ddn get cloud-environment-variable 

## Synopsis
See docs here: https://hasura.io/docs/3.0/ci-cd/secrets/.

```bash
$ ddn get cloud-environment-variable --help
See docs here: https://hasura.io/docs/3.0/ci-cd/secrets/

Usage:
  ddn get cloud-environment-variable [flags]

Aliases:
  cloud-environment-variable, CloudEnvironmentVariable, cloudenvvar, cloudenvironmentvariable

Examples:
 # Get a Cloud Environment Variable
   ddn get cloud-environment-variable <key>
 
 # List all Cloud Environment Variables
   ddn get cloud-environment-variable 

 # Get the value of a Cloud Environment Variable in a subgraph (across all environments) of a project
   ddn get cloud-environment-variable <key> --subgraph <subgraph-name>

 # Get the the value of a Cloud Environment Variable in an environment and subgraph
   ddn get cloud-environment-variable <key> --subgraph <subgraph-name> --environment <env-name>


Flags:
  -e, --environment string   Environment of Cloud Environment Variable
  -h, --help                 help for cloud-environment-variable
  -n, --subgraph string      Subgraph of Cloud Environment Variable

Global Flags:
      --dir string                   DDN project directory in which hasura.yaml is present (default ".")
      --log-level string             Log level. accepts DEBUG, WARN, INFO, ERROR, FATAL (default "INFO")
      --no-prompt                    Set this flag to no prompt for anything, the command fails if any input is missing
  -o, --out string                   Format in which the output should be printed, accepts table, json and yaml (default "table")
  -p, --project string               DDN Project Name
  -s, --supergraph-manifest string   SupergraphManifest to use. (Optional. DefaultSupergraphManifest used  if not provided)
```
