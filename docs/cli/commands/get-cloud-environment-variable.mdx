# Hasura DDN CLI: ddn get cloud-environment-variable 

## Synopsis
List resources or get details of a resource from Hasura DDN.

```bash
$ ddn get cloud-environment-variable --help
List resources or get details of a resource from Hasura DDN

Usage:
  ddn get [command]

Examples:
 # List all your Projects
   ddn get project
		
 # Get details of a specific Project
   ddn get project <project-name>


Available Commands:
  connector-build  List Connector Builds or get details of a specific one
  project          List projects or get details of a specific one
  region           List available regions for a Project
  subgraph         List Subgraphs or get details of a specific one
  supergraph-build List SupergraphBuilds or get details of a specific one

Flags:
  -h, --help   help for get

Global Flags:
      --dir string                   DDN project directory in which hasura.yaml is present (default ".")
      --log-level string             Log level. accepts DEBUG, WARN, INFO, ERROR, FATAL (default "INFO")
      --no-prompt                    Set this flag to no prompt for anything, the command fails if any input is missing
  -o, --out string                   Format in which the output should be printed, accepts table, json and yaml (default "table")
  -p, --project string               DDN Project Name
  -s, --supergraph-manifest string   SupergraphManifest to use. (Optional. DefaultSupergraphManifest used  if not provided)

Use "ddn get [command] --help" for more information about a command.
```
