# Hasura DDN CLI: ddn update connector-manifest

## Synopsis
Update the build context of a ConnectorManifest.

```bash
$ ddn update connector-manifest --help
Update the build context of a ConnectorManifest.

Usage:
  ddn update connector-manifest <name> [flags]

Aliases:
  connector-manifest, cm, ConnectorManifest

Examples:
 # Update ConnectorManifest "mydb" in Subgraph "app"
   ddn update connector-manifest mydb --subgraph app

 # Update ConnectorManifest "mydb" in Subgraph "app" and specify values for environment variables in it's ConnectorManifest
   ddn update connector-manifest mydb --subgraph app -e ENV_KEY_1=env-value-1 -e ENV_KEY_2=env-value-2


Flags:
      --env strings       Env vars
  -h, --help              help for connector-manifest
  -s, --subgraph string   Subgraph name

Global Flags:
      --dir string                   DDN Project directory in which hasura.yaml is present (default ".")
      --log-level string             Log level. Can be DEBUG, WARN, INFO, ERROR, or FATAL. (default "INFO")
      --no-prompt                    Do not prompt for required but missing flags
      --out string                   Output format. Can be table, json or yaml. (default "table")
  -p, --project string               DDN Project name
      --supergraph-manifest string   SupergraphManifest to use
```
