# Hasura DDN CLI: ddn update connector-manifest 

## Synopsis
Update a connector's BuildContext.

```bash
$ ddn update connector-manifest --help
Update a connector's BuildContext

Usage:
  ddn update connector-manifest [connector name] [flags]

Aliases:
  connector-manifest, cm, ConnectorManifest, connectormanifest

Examples:
 # Update a connector's BuildContext using the ConnectorManifest defined in SupergraphManifest
   ddn update connector-manifest <connector-name>  --subgraph <subgraph-name>

 # Update a connector using a different ConnectorManifest
   ddn update connector-manifest <connector-name>  --subgraph <subgraph-name> --connector-manifest <connector-manifest.hml>

 # Update a connector and specify values for Env vars in the ConnectorManifest
   ddn update connector-manifest <connector-name>  --subgraph <subgraph-name> -e ENV_KEY_1=env-value-1 -e ENV_KEY_2=env-value-2


Flags:
  -c, --connector-manifest string   ConnectorManifest name (Optional)
  -e, --env strings                 env vars
  -h, --help                        help for connector-manifest
      --subgraph string             Subgraph name

Global Flags:
      --dir string                   DDN project directory in which hasura.yaml is present (default ".")
      --log-level string             Log level. accepts DEBUG, WARN, INFO, ERROR, FATAL (default "INFO")
      --no-prompt                    Set this flag to no prompt for anything, the command fails if any input is missing
  -o, --out string                   Format in which the output should be printed, accepts table, json and yaml (default "table")
  -p, --project string               DDN Project Name
  -s, --supergraph-manifest string   SupergraphManifest to use. (Optional. DefaultSupergraphManifest used  if not provided)
```
