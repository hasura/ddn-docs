# Hasura DDN CLI: ddn update connector-link 

## Synopsis
Updates the schema of a DataConnectorLink.

```bash
$ ddn update connector-link --help
Updates the schema of a DataConnectorLink

Usage:
  ddn update connector-link [connector name] [flags]

Aliases:
  connector-link, cl, ConnectorLink, connectorlink

Examples:
 # Update the schema of a DataConnectorLink
   ddn update connector-link <connector-name> --subgraph <subgraph-name>

 # Update the schema of a DataConnectorLink using the running connector URL referenced using an env var CONNECTOR_URL
   ddn update connector-link <connector-name> --subgraph <subgraph-name> -e CONNECTOR_URL=https://connector-host:post

 # Update the schema of a DataConnectorLink while adding all the Models and Commands from the updated schema to the project
   ddn update connector-link <connector-name> --subgraph <subgraph-name> --add-all-resources


Flags:
      --add-all-resources   Add all Models and Commands from the updated DataConnectorLink to the local Hasura project
  -h, --help                help for connector-link
      --subgraph string     Subgraph name

Global Flags:
      --dir string                   DDN Project directory in which hasura.yaml is present (default ".")
      --log-level string             Log level. Can be DEBUG, WARN, INFO, ERROR, or FATAL. (default "INFO")
      --no-prompt                    Do not prompt for required but missing flags
  -o, --out string                   Output format. Can be table, json or yaml. (default "table")
  -p, --project string               DDN Project name
  -s, --supergraph-manifest string   SupergraphManifest to use.
```
