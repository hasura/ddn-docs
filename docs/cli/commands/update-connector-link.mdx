# Hasura DDN CLI: ddn update connector-link 

## Synopsis
Updates the schema of a DataConnectorLink.

```bash
$ ddn update connector-link --help
Updates the schema of a DataConnectorLink

Usage:
  ddn update data-connector-link <name> [flags]

Aliases:
  data-connector-link, DataConnectorLink, cl, connector-link

Examples:
 # Update the schema of a DataConnectorLink 'mydb'
   ddn update data-connector-link mydb

 # Update the schema of a DataConnectorLink 'mydb' using the URL referenced via env var CONNECTOR_URL
   ddn update data-connector-link mydb -e CONNECTOR_URL=https://connector-host:post

 # Update the schema of a DataConnectorLink 'mydb' and add all Models and Commands to the local metadata
   ddn update data-connector-link mydb --add-all-resources


Flags:
      --add-all-resources   Add all Models and Commands from the updated DataConnectorLink to the local Hasura project
  -h, --help                help for data-connector-link
      --subgraph string     Subgraph name

Global Flags:
      --dir string                   DDN Project directory in which hasura.yaml is present (default ".")
      --log-level string             Log level. Can be DEBUG, WARN, INFO, ERROR, or FATAL. (default "INFO")
      --no-prompt                    Do not prompt for required but missing flags
      --out string                   Output format. Can be table, json or yaml. (default "table")
  -p, --project string               DDN Project name
      --supergraph-manifest string   SupergraphManifest to use
```
