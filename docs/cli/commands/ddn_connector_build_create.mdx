# DDN CLI: ddn connector build create

Create a ConnectorBuild on Hasura DDN.

## Synopsis

Create a ConnectorBuild on Hasura DDN

```bash
ddn connector build create --connector <path-to-connector-config-file> [flags]
```

## Examples

```bash
# Create ConnectorBuild from Connector located at ./foo/my_db.connector.cloud.yaml
 ddn connector build create --connector ./foo/my_db.connector.cloud.yaml --project pet-lion-2649

# Create ConnectorBuild from Connector located at ./foo/my_db.connector.cloud.yaml with global and regional environment variables
 ddn connector build create --connector ./foo/my_db.connector.cloud.yaml --env key1=val1 --region-env gcp-europe-west1:key2=val2

# Create ConnectorBuild from Connector located at ./foo/my_db.connector.cloud.yaml with global and regional environment variable files
 ddn connector build create --connector ./foo/my_db.connector.cloud.yaml --env-file ./env --region-env-file gcp-europe-west1:./region-1/.env

# Create ConnectorBuild from Connector located at ./foo/my_db.connector.cloud.yaml and update schema of DataConnectorLink "mydb"
 ddn connector build create --connector ./foo/my_db.connector.cloud.yaml --target-connector-link mydb --target-subgraph ./subgraph.yaml --project pet-lion-2649
```

## Options

```sass
-c, --connector string               Path to Connector config file
-e, --env stringArray                Environment variable, e.g. key=val. Can be repeated to provide multiple env vars
    --env-file stringArray           Path to .env file. Can be repeated to provide multiple env files
-h, --help                           help for create
-p, --project string                 DDN Project name
    --region-env stringArray         Region environment variable. Should be prefixed with the region followed by a colon e.g. "gcp-asia-south1:key=val". Can be repeated to provide multiple region env vars
    --region-env-file stringArray    Path to Connector region .env file. Should be prefixed with the region followed by a colon e.g. "gcp-asia-south1:./region2/.env". Can be repeated to provide multiple region env files
    --target-connector-link string   DataConnectorLink to update with the schema from the ConnectorBuild
    --target-env-file string         Path to the env file in which the Build URLs should be updated in
    --target-subgraph string         Path to Subgraph config file containing target DataConnectorLink
```
