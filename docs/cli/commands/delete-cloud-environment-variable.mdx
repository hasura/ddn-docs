# Hasura DDN CLI: ddn delete cloud-environment-variable 

## Synopsis
See docs here: https://hasura.io/docs/3.0/ci-cd/secrets/.

```bash
$ ddn delete cloud-environment-variable --help
See docs here: https://hasura.io/docs/3.0/ci-cd/secrets/

Usage:
  ddn delete cloud-environment-variable [flags]

Aliases:
  cloud-environment-variable, CloudEnvironmentVariable, cloudenvvar, cloudenvironmentvariable

Examples:
 # Delete a Cloud Environment Variable
   ddn delete cloud-environment-variable <key> --environment <env-name>


Flags:
  -e, --environment string   Environment where the Cloud Environment Variable can be referenced (required)
  -h, --help                 help for cloud-environment-variable
  -n, --subgraph string      Subgraph where the Cloud Environment Variable can be referenced (required)

Global Flags:
      --dir string                   DDN project directory in which hasura.yaml is present (default ".")
      --log-level string             Log level. accepts DEBUG, WARN, INFO, ERROR, FATAL (default "INFO")
      --no-prompt                    Set this flag to no prompt for anything, the command fails if any input is missing
  -o, --out string                   Format in which the output should be printed, accepts table, json and yaml (default "table")
  -p, --project string               DDN Project Name
  -s, --supergraph-manifest string   SupergraphManifest to use. (Optional. DefaultSupergraphManifest used  if not provided)
```
