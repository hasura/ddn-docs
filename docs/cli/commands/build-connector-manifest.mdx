# Hasura DDN CLI: ddn build connector-manifest 

## Synopsis
Build and deploy a data connector using its ConnectorManifest.

```bash
$ ddn build connector-manifest --help
Build and deploy a data connector using its ConnectorManifest

Usage:
  ddn build connector [connector name] [flags]

Aliases:
  connector, connector-manifest, ConnectorManifest, connectormanifest, cm

Examples:
 # Build a cloud connector and wait for deployment to be done
  ddn build connector <connector-name> --subgraph <subgraph-name> --wait


Flags:
      --connector-manifest string   ConnectorManifest to use. By default, the one specified in the SupergraphManifest is used
  -e, --env strings                 env vars
  -h, --help                        help for connector
      --subgraph string             name of the Subgraph
      --wait                        blocks the process till the connector build is done (Applicable only for cloud)
      --wait-timeout int            Max wait time in seconds to wait for connector deployment. Default: 60 (default 60)

Global Flags:
      --dir string                   DDN project directory in which hasura.yaml is present (default ".")
      --log-level string             Log level. accepts DEBUG, WARN, INFO, ERROR, FATAL (default "INFO")
      --no-prompt                    Set this flag to no prompt for anything, the command fails if any input is missing
  -o, --out string                   Format in which the output should be printed, accepts table, json and yaml (default "table")
  -p, --project string               DDN Project Name
  -s, --supergraph-manifest string   SupergraphManifest to use. (Optional. DefaultSupergraphManifest used  if not provided)
```
