# DDN CLI: ddn supergraph build local

Build the Supergraph and generate assets to run the local Engine. (Uses the DDN hosted metadata build service).

## Synopsis

Build the Supergraph and generate assets to run the local Engine. (Uses the DDN hosted metadata build service)

```bash
ddn supergraph build local --output-dir <path-to-engine-directory> [flags]
```

## Examples

```bash
# Build the Supergraph and output it to engine directory
 ddn supergraph build local --output-dir <path-to-engine-directory> --supergraph supergraph.local.yaml

# Build the Supergraph and output it to engine directory with a custom subgraph env file and env vars for Subgraph "myapp
 ddn supergraph build local --output-dir <path-to-engine-directory> --supergraph supergraph.local.yaml --subgraph-env-file myapp:./myapp/.env.myapp.local --subgraph-env myapp:foo=bar
```

## Options

```sass
-h, --help                            help for local
    --output-dir string               Path to the engine directory to output the build artifacts (required)
    --subgraph-env stringArray        Subgraph environment variable. Should be prefixed with the Subgraph name followed by a colon e.g. "myapp:key=val". Can be repeated to provide multiple Subgraph env vars
    --subgraph-env-file stringArray   Path to Subgraph .env file. Should be prefixed with the Subgraph name followed by a colon e.g. "myapp:./myapp/.env". Can be repeated to provide multiple subgraph env files
    --supergraph string               Path to Supergraph config file
```
