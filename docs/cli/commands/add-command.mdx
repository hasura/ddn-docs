# Hasura DDN CLI: ddn add command

## Synopsis
Add a new Command to the local metadata

```bash
$ ddn add command --help
Add a new Command to the local metadata

Usage:
  ddn add command [flags]

Examples:
 # Add a new Command "Login" from the procedure "Login" in the DataConnectorLink "actions"
   ddn add command --data-connector-link actions --name Login
	 

Flags:
      --data-connector-link string   Name of the DataConnectorLink (required)
  -h, --help                         help for command
      --name string                  Name of the procedure or function to add as the Command (required)
      --subgraph string              Subgraph that contains the DataConnectorLink

Global Flags:
      --dir string                   DDN Project directory in which hasura.yaml is present (default ".")
      --log-level string             Log level. Can be DEBUG, WARN, INFO, ERROR, or FATAL. (default "INFO")
      --no-prompt                    Do not prompt for required but missing flags
      --out string                   Output format. Can be table, json or yaml. (default "table")
  -p, --project string               DDN Project name
      --supergraph-manifest string   SupergraphManifest to use
```
