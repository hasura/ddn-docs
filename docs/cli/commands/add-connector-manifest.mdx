# Hasura DDN CLI: ddn add connector-manifest 

## Synopsis
Add a new ConnectorManifest to the local metadata.

```bash
$ ddn add connector-manifest --help
Add a new ConnectorManifest to the local metadata

Usage:
  ddn add connector-manifest [connector name] [flags]

Aliases:
  connector-manifest, ConnectorManifest, connectormanifest, cm

Examples:
 # Add a new "hasura/nodejs:v1.0.0" connector
   ddn add connector-manifest <connector-name> --subgraph <subgraph-name> --hub-connector hasura/postgres:v0.5.1 --type cloud


Flags:
      --clean                  ignore local cache of the connector package
  -h, --help                   help for connector-manifest
      --hub-connector string   name and version of connector in Hasura Connector Hub
      --subgraph string        name of the Subgraph to add the connector to
      --type string            one of cloud | endpoints . whether the connector should be deployed on Hasura DDN or is already running. Choose 'cloud' by default

Global Flags:
      --dir string                   DDN Project directory in which hasura.yaml is present (default ".")
      --log-level string             Log level. Can be DEBUG, WARN, INFO, ERROR, or FATAL. (default "INFO")
      --no-prompt                    Do not prompt for required but missing flags
  -o, --out string                   Output format. Can be table, json or yaml. (default "table")
  -p, --project string               DDN Project name
  -s, --supergraph-manifest string   SupergraphManifest to use.
```
