# Hasura DDN CLI: ddn add connector-manifest 

## Synopsis
Add a new Connector Manifest to the local metadata.

```bash
$ ddn add connector-manifest --help
Add a new Connector Manifest to the local metadata

Usage:
  ddn add connector-manifest [connector name] [flags]

Aliases:
  connector-manifest, ConnectorManifest, connectormanifest, cm

Examples:
 # Add a new hub connector hasura/sendgrid:v0.5.0 of type cloud to subgraph 'app'
   ddn add connector-manifest <connector-name> --subgraph app --dir <project_dir>  --hub hasura/sendgrid:v0.5.0 --type cloud

Flags:
      --clean             ignore local cache of the connector package
  -h, --help              help for connector-manifest
      --hub string        name and version of connector in hasura hub
      --subgraph string   name of the subgraph to add the connector to
      --type string       one of cloud | local | endpoint . This determines how the connector will be deployed. If unsure, choose 'cloud'

Global Flags:
      --dir string                   DDN project directory in which hasura.yaml is present (default ".")
      --log-level string             Log level. accepts DEBUG, WARN, INFO, ERROR, FATAL (default "INFO")
      --no-prompt                    Set this flag to no prompt for anything, the command fails if any input is missing
  -o, --out string                   Format in which the output should be printed, accepts table, json and yaml (default "table")
  -p, --project string               DDN Project Name
  -s, --supergraph-manifest string   Supergraph manifest to use. (Optional. DefaultSupergraphManifest used  if not provided)
```
