# Private Endpoints

:::tip TL;DR
Private Endpoints allow secure, private connectivity between your infrastructure and Hasura's Data Planes without exposing traffic to the public internet. Currently supported on AWS, with Azure and GCP support coming soon.
:::

## Introduction

Private Endpoints in Hasura Private DDN provide secure, private connectivity between your infrastructure and Hasura's Data Planes without exposing traffic to the public internet. This feature is essential for enterprises with strict security, compliance, and data privacy requirements.

## Benefits of Private Endpoints

- **Enhanced Security**: Traffic never traverses the public internet, reducing attack surface
- **Simplified Network Architecture**: No need for VPN or Direct Connect
- **Reduced Latency**: Uses cloud provider's internal network infrastructure
- **Scalability**: Can handle high throughput workloads
- **Compliance**: Helps meet regulatory requirements for data privacy and security

## Supported Cloud Providers

Private Endpoints are currently supported on:

- **AWS**: Using AWS PrivateLink
- **Azure**: Using Azure Private Link (coming soon)
- **GCP**: Using Private Service Connect (coming soon)

## Best Practices

- Create endpoints in multiple availability zones for high availability
- Use descriptive names for your endpoints to easily identify their purpose
- Regularly review and audit your endpoints for security and compliance
- Consider implementing additional security measures like security groups and network ACLs
