# Hasura Data Domain Specification Models

## Introduction

Models in DDS refer to a collection of objects (otherwise known as rows or documents) of a given DDS type. Models are
backed by a data source and can support CRUD operations.

## Description

To create a model, you need to define a DDS object of kind `model`, which has both `name` and `dataType` fields.

### Metadata structure

```json
{
  "kind": "model",
  "name": <ModelName>,
  "dataType": <TypeName>,
  "arguments": [
    {
      "name": "arg1",
      "type": <Type>
    }
  ],
  "globalIDSource": true | false
}
```

| Field       | Type   | Description              |
| ----------- | ------ | ------------------------ |
| `ModelName` | String | Name of the model.       |
| `TypeName`  | String | Name of the object type. |
| `arguments` | Array  | List of arguments.       |

## Examples

```json
{
  "kind": "model",
  "name": "ArticlesByAuthor",
  "dataType": "article",
  "arguments": [
    {
      "name": "author_id",
      "type": "Int"
    }
  ]
}
```
