# Hasura Data Domain Specification Models

## Introduction
Models in DDS refer to a collection of objects (a.k.a. rows or documents) of a
given DDS type. Models are backed by a data source and can support CRUD
operations.

## Description

To create a model you need to define a DDS object of kind `model`. Which has `name` and `dataType`.

### Metadata Structure

```json
{
  "kind": "model",
  "name": <ModelName>,
  "dataType": <TypeName>,
  "arguments": [
    {
      "name": "arg1",
      "type": <Type>
    }
  ],
  "globalIDSource": true | false
}
```

- `ModelName`: Name of the model in string
- `TypeName`: Name of the object type in string
- `arguments`: Optional field. List of arguments.

## Examples

```json
{
  "kind": "model",
  "name": "ArticlesByAuthor",
  "dataType": "article",
  "arguments": [
    {
      "name": "author_id",
      "type": "Int"
    }
  ]
}
```
