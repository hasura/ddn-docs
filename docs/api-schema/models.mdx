# Hasura Data Domain Specification Models

## Introduction

Models in DDS refer to a collection of objects (otherwise known as rows or documents) of a given DDS type. Models are
backed by a data source and can support CRUD operations.

## Description

To create a model, you need to define a DDS object of kind `model`, which has both `name` and `dataType` fields.

### Metadata structure

```json
{
  "kind": "model",
  "name": "<ModelName>",
  "dataType": "<TypeName>",
  "arguments": [
    {
      "name": "arg1",
      "type": <Type>
    }
  ],
  "globalIDSource": true | false
}
```

| Field          | Type         | Required | Description                                                         |
|----------------|--------------|----------|---------------------------------------------------------------------|
| name           | `String`     | true     | Name of the model.                                                  |
| dataType       | `String`     | true     | Name of the object type.                                            |
| arguments      | `[Argument]` | false    | List of arguments.                                                  |
| globalIDSource | `Boolean`    | false    | If [Global ID](./global-id.mdx) should be generated for this model. |

#### Argument

| Field | Type     | Required | Description                                                                                                                                 |
|-------|----------|----------|---------------------------------------------------------------------------------------------------------------------------------------------|
| name  | `String` | true     | Name of the argument                                                                                                                        |
| type  | `Type`   | true     | Type of the argument. String in case of primitive or custom types. An object in the case of container types. (Link to JSONSchema of `Type`) |

## Examples

```json
{
  "kind": "model",
  "name": "ArticlesByAuthor",
  "dataType": "article",
  "arguments": [
    {
      "name": "author_id",
      "type": "Int"
    }
  ]
}
```
