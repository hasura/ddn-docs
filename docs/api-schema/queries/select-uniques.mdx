# Select Uniques

## Introduction

You can fetch a single object based on unique field values from a model.

query:
```graphql
query {
    ArticleByID(article_id: 1) {
        article_id title
    }
}
```
response:
```json
{
  "ArticleByID": {
    "article_id": 1,
    "title": "The Next 700 Programming Languages"
  }
}
```

## Open DDS Metadata
You need to define `selectUniques` property in [`modelGraphqlApi`](/api-schema/queries/open-dds-reference.mdx#open-dds-reference-modelgraphqlapi) open DDS object.
It is required to specify root field name of select Uniques query and unique identifier fields. See [`SelectUniqueGraphQlDefinition`](/api-schema/queries/open-dds-reference.mdx#selectuniquegraphqldefinition) reference for more details.

A sample metadata for `Articles` momdel:
```json
{
    "kind": "modelGraphqlApi",
    "modelName": "Articles",
    "selectUniques": [
        {
            "queryRootField": "ArticleByID",
            "uniqueIdentifier": [
                "article_id"
            ]
        }
    ]
}
```
The `article_id` field is serving as unique identifier.

The above metadata generates the following graphql schema.

```graphql
type Article {
  article_id: Int!
  author_id: Int!
  title: String!
}

interface Node {
  id: ID!
}

type Query {
  ArticleByID(article_id: Int!): Article
  node(id: ID!): Node
}
```
