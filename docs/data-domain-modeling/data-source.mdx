# Hasura Data Domain Specification Data Source

## Introduction

Data Source is the way to define a source of the data with the configuration and the operations that it can support. It
may be backed with a SQL database, a NoSQL database, a REST API, a GraphQL API, a file or any other data source.

including the connector URL and the schema of the data in which a data source can resolve data or react to any commands.

In the Open Data Domain Specification (OpenDD Spec), a DataSource must have the connector url and the schema of the data
in which a data source can resolve data in the shape of [Models](./models.mdx) or respond to any commands.

Once a Data Source is declared it can then be referenced on `source` configuration of a [Model](./models.mdx) to bind it
to the data domain.

## Description

To create a model, you need to define a OpenDD object with `"kind": "DataSource"`, which has `name`,
`data_connector_url` and `schema` fields.

### Metadata structure

```json
{
  "kind": "DataSource",
  "name": "<DataSourceName>",
  "data_connector_url": <DataConnectorURL>,
  "schema": "<SchemaResponse>",
}
```

| Field              | Type     | Required | Description                                                                                                            |
| ------------------ | -------- | -------- | ---------------------------------------------------------------------------------------------------------------------- |
| name               | `String` | true     | Name of the Data Source.                                                                                               |
| data_connector_url | `String` | true     | URL through which the data source connector can be accessed                                                            |
| schema             | `Object` | true     | The schema response from the data connector- includes scalar_types, object_type, collections, functions and procedures |

#### Schema Response

Schema Response is an object that defines the schema of the data source. It has the following fields:

```json
{
  "scalar_types": "<ScalarTypes>",
  "object_types": "<ObjectTypes>",
  "collections": "<Collections>",
  "functions": "<Functions>",
  "procedures": "<Procedures>"
}
```

| Field        | Type    | Required | Description                 |
| ------------ | ------- | -------- | --------------------------- |
| scalar_types | `Array` | true     | Scalar types in the schema. |
| object_types | `Array` | true     | Object types in the schema. |
| collections  | `Array` | true     | Collections in the schema.  |
| functions    | `Array` | true     | Functions in the schema.    |
| procedures   | `Array` | true     | Procedures in the schema.   |
